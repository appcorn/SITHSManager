Detailed Description

Protocol for a biometric GUI. The GUI is used to display to the user what is happening when enrolling or verifying.

- (void) displayChosenImage: (UIImage *)  image
optional

Tells the GUI to display a fingerprint image from the sensor. The image has been chosen for further processing, e.g. for enrollment or verification. The GUI decides if it wants to show the image or just simply ignore it. This function will likely be called together with a call to displayImage. Note that the image in the call to displayImage may be different from the image in the call to displayChosenImage, but more likely they will be the same image.

Parameters:
[in]imageis the image from the sensor to be displayed.
- (void) displayChosenRawImage: (PBImageRaw *)  image
optional

This behaves exactly as displayChosenImage: but the provided image has not been processed for enhanced viewing purpose, i.e. this is the raw image taken from the sensor. Note that both the image data (image->pixels) and the image struct must be freed by calls to free() in that particular order, i.e. free(image->pixels); free(image);

Parameters:
[in]imageis the raw image from the sensor.
- (void) displayEvent: (PBEvent)  event_
forFinger: (PBBiometryFinger *)  finger 
optional

Tells the GUI to display to the user that an event has occurred, for possible events, see PBEvent. The GUI chooses what will happen for each event, it could e.g. display a text or image for the user indicating what happened, light a LED or simply just ignore the event.

Parameters:
[in]event_is the type of event, see PB_EVENT_X.
[in]fingeris the finger in case of the events PBEventPromptSwipeFinger and PBEventAlertFingerRejected. For other events the finger will be nil.
- (void) displayImage: (UIImage *)  image
optional

Tells the GUI to display a fingerprint image from the sensor. The GUI decides if it wants to show the image or just simply ignore it. This function will most likely be called together with a posting of the event PBEventAlertImageCaptured.

Parameters:
[in]imageis the image from the sensor to be displayed.
- (void) displayQuality: (uint8_t)  imageQuality
andArea: (uint32_t)  area
imageQualityThreshold: (uint8_t)  imageQualityThreshold
areaThreshold: (uint32_t)  areaThreshold 
optional

Tells the GUI to display quality information of the captured image. The GUI may choose to show parts or all of this quality information or simply ignore it. The GUI may choose to display this information as numbers or e.g. as bars indicating the values.

Parameters:
[in]imageQualityis the image quality of the captured image. The quality values range from 0 (worst quality) to 100 (best quality).
[in]areais the fingerprint area in the captured image, in mm^2.
[in]imageQualityThresholdis the threshold that are used when checking if the image quality is acceptable or not.
[in]areaThresholdis the threshold that are used when checking if the fingerprint area is acceptable (large enough) or not.

341 /* Attribute to set/get whether or not the library automatically handles

343  Valid values are 0 to disable and 1 to enable the feature. The size of the
344  value buffer must be set to sizeof(BYTE). */
345 #define SCARD_ATTR_AUTO_BACKGROUND_HANDLING SCARD_ATTR_VALUE(SCARD_CLASS_VENDOR_DEFINED, 0xB100)
346 
347 /* Attribute to set/get whether an inserted card shall be automatically powered when inserted
348  in the reader.
349  Most desktop implementations of PCSC will automatically power on the smart when it is inserted
350  in the smart card reader. This implementation however will not automatically power on the card
351  with regards to battery life of the iOS device.
352  This means that the SCARD_READERSTATE structure used in SCardGetStatusChange will be populated
353  somewhat differently compared with a desktop implementation. For example, the ATR value in
354  rgbAtr/cbAtr will be blank prior to calls to SCardConnect.
355  It is possible to enabled automatic power on in order to mimic desktop PCSC behaviour using
356  the proprietary SCARD_ATTR_AUTO_POWER_ON attribute together with a call to SCardSetAttrib.
357  The default value is set to disabled. The set value is global and is valid until the last
358  SCARDCONTEXT is released with a call to SCardReleaseContext.
359  Valid values are 0 to disable and 1 to enable the feature. The size of the
360  value buffer must be set to sizeof(BYTE). */
361 #define SCARD_ATTR_AUTO_POWER_ON SCARD_ATTR_VALUE(SCARD_CLASS_VENDOR_DEFINED, 0xB101)
